# TinyTPU Bundle Format (wire format)

> **See also:** `doc/tinytpu_asm.md` — the TASM human-readable assembly language
> that compiles to this wire format via `scripts/tasm.py`.

The BSV testbench (`TbTinyTPURuntime.bsv`) reads integers one at a time via
the `tinytpu_bundle_read_int()` BDPI C function from the file pointed to by
`$TINYTPU_BUNDLE`.  The format is whitespace-delimited integers; `#` comment
lines are skipped.  Line breaks have no semantic meaning — only the integer
sequence matters.

| Record | Fields | Meaning |
| --- | --- | --- |
| `0` | `addr` + 16 int8 values | Load a 4x4 MXU weight tile. |
| `1` | `addr` + 4 int8 values | Load one MXU activation vector. |
| `2` | `opcode vmemAddr vregDst vregSrc vpuOp vregSrc2 mxuWBase mxuABase mxuTLen` | Load one scalar-unit instruction. |
| `3` | `flag` | Emit `mxu_result` after halt when `flag` is `1`. |
| `4` | none | End bundle loading and start execution. |
| `5` | `addr` + 16 int32 values | Load one 4x4 VMEM tile. |
| `6` | `addr` | Emit `vmem_result` for one VMEM tile after halt. |

The profiler module can parse and write these records through
`profiler.bundle.Bundle`. Use `scripts/dump_tinytpu_bundle.py` to normalize an
existing bundle or capture the bundle generated by a tinygrad script.
